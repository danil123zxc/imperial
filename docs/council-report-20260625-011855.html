Final Plan

  1. Freeze a baseline. Save current per-row results before changing row semantics, using currently supported metrics.
  2. Create the row audit table. Include row ID, behavior, lane, current and candidate file IDs, indexed status, source path, answer-key quality, action, quarantine reason, and backlog category.
  3. Repair or quarantine defects. Start with imperial-cite-003; either ingest/convert the absence-act .doc, retarget the row to real indexed evidence, or quarantine it.
  4. Backfill and rewrite in one pass. For cite rows, add concrete Russian references, file-level gold IDs, and specific hints. For conflict rows, include both competing source IDs.
  5. Split lanes. Separate indexed answerability, conflict/version behavior, refusal/out-of-corpus behavior, and known missing-document coverage.
  6. Add corpus-aware validation. Enforce row schema, ID resolution against extracted chunks, lane-specific ID rules, hint checks, quarantine semantics, and unsupported metric gating.
  7. Rebuild judge calibration. Add hard cite negatives, partial answers, wrong-role/entity cases, conflict examples, and refusal examples. Report accuracy, confusion matrix, separation, model, and prompt/config.
  8. Enable semantic correctness only after the gate. Keep factual_correctness disabled until the judge passes and the metric is either supported in Phoenix or explicitly routed through a supported Ragas runner.
  9. Expand last. Add 5-8 in-domain near-miss refusals and audit-derived coverage gaps only after the 37-row dataset is clean.
