Where the Council Agrees

The council agrees this is not 548 independent bugs. The top-level problem is an undefined static-analysis contract plus dynamic compatibility surfaces that Pylance cannot infer.

All advisors agree on these constraints: do not disable reportAttributeAccessIssue globally, do not weaken monkeypatch-heavy tests, and use local suppressions only after proving a dynamic seam or third-party stub is the real problem.

Where the Council Clashes

The only meaningful disagreement is whether to fix source contracts before or after package facades. The stronger argument is to inventory and fix facade exports first because dynamic __all__ and star exports are high-fanout noise multipliers.

Before replacing facades, inventory old import paths and monkeypatch-forwarding dependencies. Add smoke coverage for compatibility aliases so explicit exports do not silently break runtime identity or tests.

Use a ratchet: save the JSON baseline, bucket by path/rule, prevent new diagnostics, then reduce the baseline by category.

Fix Plan

  1. Stabilize analysis. Add pyrightconfig.json and optionally VS Code workspace settings. Add a non-blocking diagnostic summary command that groups JSON by path/rule/message.
  2. Inventory compatibility facades. Use repo search to list imports and monkeypatches through alias modules before changing exports. Add smoke tests for old import paths that must remain valid.
  3. Make exports static. Replace dynamic __all__ and star exports with explicit exports, TYPE_CHECKING surfaces, or .pyi stubs where alias identity must remain dynamic.
  4. Tighten source contracts. Add small protocols for compiled workflows, retrievers, vector stores, embeddings, and callables. Fix TypedDict requiredness and the concrete argument mismatches.
  5. Type test doubles. Add shared typed fake modules/classes for OpenTelemetry, Phoenix, Elasticsearch, provider clients, and callable counters. Use cast(Any, ...) only at deliberate dynamic seams.
  6. Ratchet enforcement. Keep full Pyright advisory initially, gate no-new-errors, then require clean src, then include scripts and finally tests.
