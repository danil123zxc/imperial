LLM Council · Verdict

Should the Imperial RAG eval be hardened "quality-only, no new questions"?

2026-06-25 · 5 advisors · 5 peer reviews · 1 chairman · follow-up to the 2026-06-23 council
The question Imperial RAG is a Russian-language RAG over a company's internal operational docs (warehouse/HR/logistics/pricing; 103 docs) where wrong answers have operational consequences. The eval is 37 questions (27 cite / 5 conflict / 5 refuse) plus a 20-row Russian judge-calibration set. The user locked a quality-only plan — rewrite vague reference answers into concrete facts (W1), backfill gold context IDs (W2), turn on factual_correctness (W3), rebuild calibration quality (W4), tighten source hints (W5), add a validation lint (W6) — and explicitly no new questions. Is that the right call, and what's the biggest flaw?

Verdict Do W1–W6, but change two things

"Quality-only, no new questions" is the wrong strategy — and the vote was unanimous. All five advisors and all five reviewers rejected the constraint. The six workstreams are real work and none is wasted, but they sharpen precision when the eval's actual problem is bias: a 73%-happy-path sample, polished, becomes a false green — a dashboard that reads "validated" while the operationally dangerous cases were never tested.

🔩 The load-bearing catch (every reviewer, no advisor): the judge itself is unvalidated. factual_correctness (W3) is only as trustworthy as a judge calibrated on trivial polar-opposite negations — and such a judge can't be trusted to grade the harder cases everyone wants to add. W4 is the precondition, not the fourth step.

Two changes that make the plan trustworthy

1 · Reorder around the judge — W4 first. Rebuild the calibration set with hard/borderline negatives, then validate it against a small human-graded held-out set (measure agreement). Keep factual_correctness (W3) OFF until that lands. Correct order: W4 → validate judge → W3 → harder cases.

2 · Crack the lock exactly once — in-domain near-miss refusals. Not a question factory (the "1 fact = 3 questions" math fails for conflict/refuse, which need engineered conditions). Add ~5–8 plausible, in-domain questions that should refuse (e.g. "return window for [scenario not in corpus]?"). Reframe the rule as "no new document families," not "no new questions."

Do first  Rebuild + human-validate the judge (W4). Hand-grade ~15 answers and check whether the calibrated judge agrees with the humans. Everything downstream produces a number only as trustworthy as that judge.

Chairman's 30-minute aside: first check whether production query logs exist. A bias problem is solved by sampling the real question distribution — if logs exist, they outrank every invented question here.

Advisor responses
The Contrarian▶

Looks for the fatal flaw.

Fixing measurement quality of 37 questions only makes the eval accurate about a tiny, biased sample — and a sharper instrument on the happy path manufactures false confidence. The worst call is W3+W1 together: factual_correctness graded only where the system already answers measures paraphrasing of handed facts, never the harmful mode — confidently answering an in-domain question it should refuse, or the wrong-entity approver. Cheapest high-value add: the in-domain near-miss refusal ("return window for X" where X isn't covered) — the only test separating a careful RAG from a confident liar. Do W1–W6, but un-lock that one.

The First Principles Thinker▶

Rebuilds the problem from scratch.

An eval is a sampling estimator of production risk; trustworthiness is dominated by coverage of the failure distribution, not polish per probe. W1–W6 are all fidelity-per-question; none touch coverage — so the constraint optimizes precision while the dominant error is bias → a false green, worse than a smoke test. W4 adds hard negatives to calibration while the plan refuses to give the system hard cases — incoherent. Reframe: not "no new questions" but "no new document families"; adversarial variants of the 27 covered docs cost almost nothing.

The Outsider▶

Fresh eyes, zero context.

Treating "no new questions" as a virtue is the bug. Polishing 37 is like a restaurant that failed inspection for having 3 dishes deciding to "make those 3 dishes really good" — the dishes were never the problem, the menu is. The 6 fixes make the eval more rigorous about the same happy paths, not more representative. ~10 document families have zero signal. And W1 is the expensive, judgment-heavy part — once you're reading every doc closely, 2 new questions per family is nearly free.

The Expansionist▶

Hunts the upside everyone's missing.

W1 is question-generation in disguise. Mining each gradeable fact is 90% of the cost of writing new questions: one fact = one reference answer or three questions. Same labor, 3× the eval. The corpus is a moat; the eval should compound with it, not certify a frozen 37-row snapshot. W1+W2 build the infrastructure to auto-generate and grade at scale; W4's calibrated judge is reusable across an unbounded set. "Trustworthy" and "37 questions" cannot coexist.

The Executor▶

Only asks: what do you do Monday morning?

W1 is most of the value and it's blocked on an unstated step — you can't rewrite references from your armchair; each needs you to open the doc and grab the exact fact, which is the same motion as W2 and W5. So do one pass per question, not six workstreams. Monday: take the 12 rows that already have context IDs and write answer + ID + hint together. Biggest risk: turning on W3 before W1/W4 → a dashboard that lies. Gate W3. Build W6's lint first so it polices the rewrite.

Peer-review highlights▶

Anonymized A–E for review; revealed here.

  • Strongest: the Contrarian (3 votes) and First Principles (1) for the most surgical, buildable critique; the Executor (1) for executability.
  • Biggest blind spot: the Executor (3 votes) — accepts the lock and optimizes the wrong objective; the Expansionist (1) — the "1 fact = 3 questions" math fails for conflict/refuse, which need engineered conditions; the Outsider (1) — diagnosis without an execution path.
  • Universal miss (all 5 reviewers): the 20-row judge-calibration set is load-bearing and unvalidated — a judge tuned on trivial negations can't grade the hard cases everyone proposes, so W4 must precede W3. Two reviewers added: anchor coverage to production query logs, not invented variants.
